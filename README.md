# J-Booking — Hotel Booking System

> **Финальный проект** стажировки.

J-Booking — сервис бронирования отелей. Всё начинается с выбора города: пользователь просматривает доступные города, переходит к списку отелей и запускает поиск с нужными фильтрами — даты заезда и выезда, количество гостей, ценовой диапазон, минимальная звёздность и набор удобств. Система возвращает только те отели, в которых есть свободные номера на указанный период. Цены отображаются в выбранной валюте — UZS, USD или RUB.

Когда номер выбран, запускается процесс бронирования. Он состоит из трёх шагов: сначала номер переходит в статус **HOLD** — он зарезервирован, но ещё не подтверждён. Если в течение 3 минут пользователь не подтверждает бронирование, оно автоматически отменяется. После подтверждения статус меняется на **CONFIRMED** — у пользователя есть 1 минута, чтобы перейти к оплате. Если оплата не поступает — бронирование сбрасывается обратно в HOLD. При успешной оплате статус становится **PAID**: сервис обращается к внешнему **Transaction Processing Service** для проведения транзакции, а затем через **Notification Service** отправляет уведомление гостю.

Если планы изменились — бронирование можно отменить в любой момент. Сервис автоматически рассчитает сумму возврата: при отмене более чем за 72 часа до заезда возвращается 100%, от 24 до 72 часов — 50%, менее чем за 24 часа — возврат не предусмотрен.

Помимо бронирования, гости могут оставлять отзывы об отелях и просматривать чужие. Для администраторов предусмотрен отдельный эндпоинт для обновления цен на номера.

## Жизненный цикл бронирования

```
POST /bookings  →  HOLD  →  POST /bookings/confirm  →  CONFIRMED  →  POST /bookings/pay  →  PAID
                    │                                      │                    ↓
                    │                ┌──────────────────────┘       Transaction Processing Service
                    │                ↓                               Notification Service
                    │        таймаут 1 мин
                    │        (Scheduler: CONFIRMED → HOLD)
                    │
                    └── таймаут 3 мин
                        (Scheduler: HOLD → CANCELLED)

PAID  →  PUT /bookings/{id}/cancel  →  CANCELLED  (возврат по политике отмены)
```

| Статус      | Описание                                                                        |
|-------------|---------------------------------------------------------------------------------|
| `HOLD`      | Номер удержан, ожидает подтверждения. Таймаут: 3 мин → автоотмена              |
| `CONFIRMED` | Бронирование подтверждено, ожидает оплаты. Таймаут: 1 мин → сброс в `HOLD`    |
| `PAID`      | Оплата прошла, уведомление отправлено. Можно отменить — возврат по политике     |
| `CANCELLED` | Бронирование отменено                                                           |

## Описание

J-Booking — REST API приложение для бронирования номеров в отелях. Реализует полный цикл бронирования: от инициализации до оплаты, с поддержкой отмены и политики возврата средств.

| Функциональность                  | Метод    | Эндпоинт                                  | Описание                                                                                  |
|-----------------------------------|----------|-------------------------------------------|-------------------------------------------------------------------------------------------|
| Просмотр городов                  | `GET`    | `/api/v1/cities`                          | Список всех городов (пагинация)                                                           |
| Список отелей по городу           | `GET`    | `/api/v1/hotels`                          | Список отелей по `cityId` (без комнат)                                                    |
| Поиск отелей                      | `GET`    | `/api/v1/hotels/search`                   | Поиск с фильтрами: даты, кол-во гостей, город, цена, звёзды, удобства (без комнат)       |
| Просмотр номеров отеля            | `GET`    | `/api/v1/rooms`                           | Список номеров по `hotelId` и `cityId`, с поддержкой валюты                               |
| Инициализация бронирования        | `POST`   | `/api/v1/bookings`                        | Создать бронирование, номер переходит в статус `HOLD`                                     |
| Подтверждение бронирования        | `POST`   | `/api/v1/bookings/confirm`                | Подтвердить бронирование, статус → `CONFIRMED`                                            |
| Оплата бронирования               | `POST`   | `/api/v1/bookings/pay`                    | Оплатить бронирование через Transaction Processing Service, статус → `PAID` + уведомление |
| Получить бронирование             | `GET`    | `/api/v1/bookings/{id}`                   | Получить бронирование по ID с поддержкой валюты                                           |
| Мои бронирования                  | `GET`    | `/api/v1/bookings/my`                     | Список бронирований гостя по PINFL (пагинация)                                            |
| Отмена бронирования               | `PUT`    | `/api/v1/bookings/{id}/cancel`            | Отменить бронирование с автоматическим расчётом возврата                                  |
| Информация о возврате             | `GET`    | `/api/v1/bookings/{id}/refund-info`       | Узнать сумму возврата до отмены                                                           |
| Создать отзыв                     | `POST`   | `/api/v1/hotels/{hotelId}/reviews`        | Оставить отзыв об отеле                                                                   |
| Список отзывов                    | `GET`    | `/api/v1/hotels/{hotelId}/reviews`        | Получить отзывы об отеле (пагинация)                                                      |
| Удалить отзыв                     | `DELETE` | `/api/v1/hotels/{hotelId}/reviews/{id}`   | Удалить свой отзыв                                                                        |
| Обновление цены номера (admin)    | `PATCH`  | `/api/v1/update-room-price`               | Обновить цену номера                                                                      |

Приложение взаимодействует с двумя внешними сервисами:
- **Transaction Processing Service** — обработка платёжных транзакций
- **Notification Service** — отправка уведомлений гостям

## Внешние сервисы (Feign клиенты)

### Transaction Processing Service
```
URL: http://localhost:7070
POST /api/v1/transactions
```
Используется при оплате подтверждённого бронирования (`/api/v1/bookings/pay`). Сервис выполняет списание средств и возвращает результат транзакции.

### Notification Service
```
URL: http://217.29.121.129/notification_api
POST /api/notifications/sending
```
Используется после успешной оплаты и при отмене бронирования. Отправляет уведомление главному гостю.

Настройка URL выполняется в `application.yml`:
```yaml
spring.cloud.openfeign.client.config:
  paymentClient.url: http://localhost:7070
  notificationClient.url: http://217.29.121.129/notification_api
```

## Политика отмены и возврата

| Период до заезда  | Возврат |
|-------------------|---------|
| Более 72 часов    | 100%    |
| От 24 до 72 часов | 50%     |
| Менее 24 часов    | 0%      |

## Поддержка валют

Цены возвращаются в запрошенной валюте. По умолчанию — `UZS`.

| Валюта | Курс (за 1 единицу в сумах) |
|--------|-----------------------------|
| UZS    | базовая                     |
| USD    | 12 000 сум                  |
| RUB    | 150 сум                     |

Параметр `currency` поддерживается в эндпоинтах: `GET /bookings/{id}`, `GET /bookings/my`, `GET /rooms`.

---

## Фоновые задачи (Scheduler)

- **BookingHoldScheduler** — каждые 60 секунд проверяет бронирования в статусе `HOLD`. Если таймаут (3 мин) истёк — переводит в `CANCELLED`.
- **BookingConfirmedScheduler** — каждые 30 секунд проверяет бронирования в статусе `CONFIRMED`. Если таймаут оплаты (1 мин) истёк — возвращает обратно в `HOLD`.
